## Goal

An "Assistant settings" panel in the Assistant tab subheader (right of the space-filter chips) hosting: model list management, modes management (define reasoning levels, future categories), a read-only composition viewer, the skills manager form, and the system-prompt editor (with reset-to-shipped).

## Why

The user needs a single surface to configure the assistant's behavior without editing `.env` or files by hand: pick models, set reasoning levels, see what the model actually sees each turn, manage skills, and tune the base prompt — all live, persisted to `instance/assistant/`.

## Scope (TDD-first)

### Failing tests first (`tests/test_assistant_settings_routes.py` — new file; `tests/test_assistant_settings_panel.py` — new)

1. `test_get_settings_returns_full_state` — `GET /api/assistant/settings` returns `{models: [...], reasoning_levels: [...], skills: [...], system_prompt: {source: 'instance'|'bundled', last_modified: iso, body: str}, composition: {layers: [...]}}`.
2. `test_put_models_writes_json` — `PUT /api/assistant/models` with `["gpt-4o", "claude-3-7-sonnet"]` writes `instance/assistant/models.json` and returns the new list. Rejects empty list (validation: at least one model).
3. `test_put_reasoning_levels_writes_json` — `PUT /api/assistant/reasoning-levels` writes `instance/assistant/reasoning.json`; validates non-empty.
4. `test_put_system_prompt_writes_override` — `PUT /api/assistant/system-prompt` with body text writes `instance/assistant/system.md`; returns `{source: 'instance', last_modified}`.
5. `test_delete_system_prompt_resets_to_shipped` — `DELETE /api/assistant/system-prompt` removes the instance override; returns `{source: 'bundled'}`.
6. `test_get_system_prompt_reset_to_exported` — the endpoint body reflects the shipped `chat/prompts/system.md` when no override exists.
7. `test_post_skill_create_calls_skill_function` — `POST /api/assistant/skills` with `{name, description, body}` calls `skills.create_skill(...)` and returns the created skill summary.
8. `test_put_skill_update_calls_function` — `PUT /api/assistant/skills/<name>` with `{body}` calls `update_skill`.
9. `test_delete_skill_route_calls_function` — `DELETE /api/assistant/skills/<name>` calls `delete_skill`; returns 404 for unknown, 400 for bundled.
10. `test_composition_viewer_layers` — the `composition.layers` field returns an ordered list: `[{kind:'base', name, last_modified}, {kind:'datetime'}, {kind:'spaces_guidance', sources:[space names]}, {kind:'tools', items:[tool names]}, {kind:'skills', items:[skill names]}]`. Built by calling the existing `build_system_prompt()` shapes (no new logic — pure assembly of what's already there).

### Implementation

- New Flask blueprint `src/routes/assistant_settings.py` (registered in `src/app.py`, `login_required`):
  - `GET /api/assistant/settings` — aggregates the full state.
  - `PUT /api/assistant/models` — writes `models.json` via `assistant_settings.write_models`.
  - `PUT /api/assistant/reasoning-levels` — writes `reasoning.json`.
  - `GET /api/assistant/system-prompt` — returns body + source + last_modified.
  - `PUT /api/assistant/system-prompt` — writes the override.
  - `DELETE /api/assistant/system-prompt` — reset to shipped.
  - `POST /api/assistant/skills`, `PUT /api/assistant/skills/<name>`, `DELETE /api/assistant/skills/<name>` — thin wrappers over `chat.skills` functions.
- Composition viewer: backend endpoint returns the layered metadata; the frontend renders cards. **No new backend logic** — it assembles what `build_system_prompt()` already produces (decompose the existing function into named layers so the viewer can label them: refactor `build_system_prompt` to return a structured `layers` list + the joined text, both derived from the same source to avoid drift).
- Frontend: `src/static/js/assistant_settings.js` (new), opens as a modal/overlay from a button in the Assistant tab subheader (next to the space-filter chips). Tabs: Model · Modes · Skills · Prompt · Composition.
- `templates/index.html`: add the subheader button + modal container + a shortcut (e.g. `Shift+,` or a button; choose one and add to the `#helpModal` table).

### Layout non-collision note

This panel opens as a **modal overlay** (transient, closes on save/escape). It must NOT collide with the Bundle-C workspace drawer (a persistent right sidebar, issue 003.09) — they live on different layers. Specify: modal z-index above the sidebar; sidebar collapses when the modal opens.
