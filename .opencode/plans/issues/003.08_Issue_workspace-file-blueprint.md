# Issue 003.08 — Workspace file blueprint (`/api/workspace/*`) (Bundle C)

- **Status**: TODO
- **PRD**: 003_PRD_deeper-assistant-integration.md (Bundle C)
- **Dependencies**: none (uses `chat/settings.py::files_dir()`)
- **Blocks**: 003.09 (sidebar view), 003.10 (attach tool + inline links)
- **Bundle**: C (foundation)

## Goal

A Flask blueprint serving the `assistant-workspace` volume (and, by design, future roots like the skills dir) with: tree listing, mkdir, upload (chunked), file download, text save, delete, move. Path-traversal-hardened. This route is ALSO the URL the model emits for inline download links (`/api/workspace/files/<root>/<path>`).

## Why

The workspace volume today is a black box — only Chainlit's internal `cl.File` layer can serve bytes, and only as a trailing attachment. The model has no sanctioned URL to deliver a file inline (it hallucinated `https://workspace.simpler.com/files/…`). The sidebar (003.09) and the model's inline links (003.10) both need real HTTP routes.

## Scope (TDD-first)

### Failing tests first (`tests/test_workspace_routes.py` — new file)

1. `test_tree_listing_workspace_root` — `GET /api/workspace/tree?root=workspace` returns the entry list (name, type, size, mtime) for `settings.files_dir()`.
2. `test_tree_listing_with_subpath` — `GET /api/workspace/tree?root=workspace&path=reports` returns the `reports/` subdir entries.
3. `test_unknown_root_rejected` — `root=foo` (not in allowlist) returns 400.
4. `test_path_traversal_rejected` — `path=../../etc/passwd` returns 400; `path=/etc/passwd` returns 400; `path=reports/../../etc/passwd` returns 400. Cover all canonical-attack variants.
5. `test_mkdir` — `POST /api/workspace/mkdir {root:'workspace', path:'reports/new'}` creates the dir; returns 201; idempotent (201 on re-create).
6. `test_upload_file` — `POST /api/workspace/upload` (multipart) with a small text file → file lands at `workspace/uploaded.txt`; returns 200 with the file URL `/api/workspace/files/workspace/uploaded.txt`.
7. `test_download_file` — `GET /api/workspace/files/workspace/reports/foo.pdf` returns the bytes with `Content-Disposition: attachment; filename="foo.pdf"` and the correct `Content-Type` (octet-stream default, `text/markdown` for `.md`).
8. `test_save_text_file` — `PUT /api/workspace/file {root:'workspace', path:'notes.md', content:'# hi'}` writes/overwrites the file.
9. `test_delete_file` — `DELETE /api/workspace/file?root=workspace&path=notes.md` removes it; returns 200.
10. `test_delete_folder_recurses` — deleting a folder path removes the subtree.
11. `test_move_file` — `POST /api/workspace/move {root:'workspace', from:'a.txt', to:'sub/a.txt'}` moves (creating `sub/` if missing).
12. `test_auth_required` — all routes return 401/redirect without login (reuse the existing `login_required` test pattern).

### Implementation

- New blueprint `src/routes/workspace.py`, registered in `src/app.py`.
- **Multi-root allowlist** (security core):
  - `ROOTS = {'workspace': lambda: settings.files_dir(), 'skills': lambda: assistant_settings.instance_skills_dir()}` — `skills` root wired here for the future file-browser-extensions (not used by the sidebar in 003.09 yet; existing as a placeholder that 003.06+future can opt into).
  - Every path arg is resolved via `os.path.realpath(os.path.join(root_dir, relpath))` then verified to start with `os.path.realpath(root_dir) + os.sep` (or be exactly the root). Reject otherwise → 400.
  - `files/<root>/<path:rest>` route serves downloads; same validation.
- Mime detection: `mimetypes.guess_type` for the common cases; default `application/octet-stream`. Force `Content-Disposition: attachment` for non-text (so browsers download, not render); inline for text/md so the sidebar can show it.
- Chunked upload: accept the existing Flask `request.files` pattern; for large files, document a max size env var `WORKSPACE_MAX_UPLOAD_MB` (default 100) and return 413 over it. (Defer streaming/chunking to a follow-up unless trivial.)
- Auth: `@login_required` on every route.
