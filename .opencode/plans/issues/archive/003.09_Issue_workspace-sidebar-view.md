## Goal

A right-drawer sidebar in the Assistant tab (~1/5 width, togglable) that browses the workspace volume via the `/api/workspace/*` blueprint: tree view, breadcrumb, drag-drop upload, create folder, create/edit `.md` inline (EasyMDE), download, delete, move (drag within tree).

## Why

The user wants direct filesystem access to the workspace the model + sandbox write to ‚Äî to drag-drop files, manage folders, edit markdown notes directly ‚Äî without leaving the Assistant tab. Fitting the project's no-build-step, vanilla-JS conventions (like `notes.js`/`mail.js`).

### Failing tests first (`tests/test_workspace_view_js.py` ‚Äî new, or extend `tests/test_routes.py`)

(JS-side tests: use the existing pattern of testing the server endpoints + a lightweight JS smoke test if the repo has one. If no JS test harness exists, test the server contract exhaustively and add a manual-test checklist for the JS interactions ‚Äî documented in acceptance criteria.)

1. `test_tree_endpoint_shape_matches_view_contract` ‚Äî the `/api/workspace/tree` response has exactly the fields `WorkspaceView` expects (name, type, size, mtime, url for files). Drives a contract test.

### Implementation

- `src/static/js/workspace.js` (new, following `NotesView`/`MailView` conventions):
  - Constructor takes a container element + the assistant-tab root; renders into a right-drawer `<aside>` injected into the Assistant-tab DOM.
  - `WorkspaceView` class: `init()`, `renderTree(node)`, `openEditor(relPath)`, `closeEditor()`, `uploadFiles(fileList, targetDir)`, `createFolder(name, parent)`, `deleteNode(relPath)`, `moveNode(fromRel, toRel)`, `download(relPath)`, `toggle()`.
  - Tree via recursive `GET /api/workspace/tree?root=workspace&path=‚Ä¶` on folder expand (lazy).
  - Breadcrumb from the current path.
  - Drag-drop: HTML5 DnD on the tree area; multi-file supported.
  - Inline `.md` editor: reuse the existing EasyMDE instance pattern from notes (lightweight new instance per open file; autosave-on-blur to `PUT /api/workspace/file`).
  - Download: anchor with `href=/api/workspace/files/workspace/<rel>` + `download` attribute.
  - Delete: confirm modal ‚Üí `DELETE`.
  - Move: drag a tree node onto another folder ‚Üí `POST /api/workspace/move`.
- `templates/index.html`:
  - Add the `WorkspaceView` drawer container in the Assistant tab section.
  - Subheader button (gear or folder icon) + shortcut `W` (when in the Assistant tab ‚Äî doesn't conflict with existing `1/2/3/4/5` + `6` Assistant; check the `#helpModal` table and add the new binding).
  - Drawer CSS: right, ~1/5 width (`min(380px, 20vw)`), togglable via a class on the body or the assistant container; persists open/closed + last-path in `localStorage`.
- `src/static/js/app.js`: wire `WorkspaceView` init when the Assistant destination is first activated; lazy (don't block chat load).
- New shortcut `W` added to the `#helpModal` table + `initKeyboardShortcuts()`.

### Layout non-collision note

This drawer is **persistent** (toggle open/closed) and lives INSIDE the Assistant tab. The Assistant settings panel (003.07) is a **modal overlay** ‚Äî it opens above the sidebar; when it opens, collapse the sidebar (or leave it; pick one and document ‚Äî **decided: modal overlays the sidebar, no collapse needed since modal covers it**).
