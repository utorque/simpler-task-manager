# Issue 003.05 — Reasoning effort as a second Mode category (Bundle B)

- **Status**: DONE (2026-07-08)
- **PRD**: 003_PRD_deeper-assistant-integration.md (Bundle B)
- **Dependencies**: 003.04 (Modes infrastructure), 003.02 (instance dir)
- **Blocks**: 003.07 (settings panel manages reasoning levels)
- **Bundle**: B

## Goal

Add a second Mode category `reasoning` (options e.g. `low/medium/high`) alongside `model`. Both appear as chat-bar pickers, switchable mid-chat. Pass the selected value to the provider as reasoning-effort, with provider-specific mapping and graceful no-op for endpoints that reject it.

## Why

The user wants reasoning effort switchable per-conversation from the chat bar "if possible". Modes supports multiple categories, so this is a natural second Mode.

## Scope (TDD-first)

### Failing tests first (`tests/test_chat_reasoning_mode.py` — new file)

1. `test_build_reasoning_mode_options` — `modes.build_reasoning_mode_options(['low','medium','high'])` returns three `ModeOption`-shaped dicts; `medium` is `default=True`.
2. `test_reasoning_levels_from_instance_json` — `available_reasoning_levels()` reads `instance/assistant/reasoning.json`; falls back to `['low','medium','high']` when absent.
3. `test_on_chat_start_sets_both_modes` — `on_chat_start` calls `set_modes` with BOTH the `model` and `reasoning` categories (extend the 003.04 test).
4. `test_current_reasoning_read_from_message` — `current_reasoning_from_modes(msg.modes, default)` returns the selected value or the default.
5. `test_provider_mappings` — `providers.apply_reasoning_effort(messages_or_params, level, provider_kind)`:
   - For `openai`, sets `reasoning_effort` key on the request params when level is non-None.
   - For `anthropic`, sets `thinking={'type':'enabled','budget_tokens':N}` (mapped low=2k, medium=8k, high=16k — tunable).
   - When level is None or `'none'`, no param is added (graceful no-op).

### Implementation

- `chat/assistant_settings.py`: add `available_reasoning_levels()` + `reasoning_json_path()` (parallel to `models.json`), default `['low','medium','high']`.
- `chat/modes.py` (from 003.04): add `REASONING_MODE_ID = 'reasoning'`, `build_reasoning_mode_options(levels)`, `current_reasoning_from_modes(modes, default)`.
- `chat/chainlit_app.py::on_chat_start`: `set_modes([*model_options, *reasoning_options])` (or the combined list shape Modes expects).
- `chat/chainlit_app.py::on_message`: read `reasoning = current_reasoning_from_modes(msg.modes, default='medium')`, pass to `get_llm()` or `run_agent()`.
- `chat/providers.py`: new `apply_reasoning_effort(request_params: dict, level: str | None, kind: str)` mutates the params dict in-place; called in `stream_chat()` before the HTTP call.
- `chat/agent.py`: thread `reasoning_effort` through if needed, or apply it in `LLMClient.stream_chat` (preferred — keep the agent loop provider-agnostic).

## Acceptance criteria

- [ ] All five failing tests pass.
- [ ] The chat bar shows two pickers: model + reasoning.
- [ ] Switching reasoning mid-chat changes the param on subsequent messages.
- [ ] A provider that returns 400 for the reasoning param is caught by the existing `except Exception as e` block in `on_message` (no crash; surface as `❌ Provider error: …`). Documented behavior.
- [ ] Default reasoning level is `medium`.

## Out of scope

- Per-model reasoning-level restrictions (all models get all levels; the provider no-ops if unsupported).
- Reasoning-level → budget tuning UI (the budgets are hardcoded constants for now).
- The settings panel to edit reasoning levels (issue 003.07 reads/writes `reasoning.json`).


## Closing note (2026-07-08)

Second Mode category `reasoning` (levels from `instance/assistant/
reasoning.json`, default `low/medium/high`, `medium` pre-selected) published
alongside `model` by `publish_modes()`. The level rides on `LLMClient`
(`get_llm(reasoning_effort=...)` per message) and is applied at the wire by
`providers.apply_reasoning_effort(params, level, kind)`: OpenAI →
`reasoning_effort=<level>`, Anthropic → `thinking={'type':'enabled',
'budget_tokens': 2048/8192/16384}`; None/'none'/unmapped-anthropic-level →
no-op. Extra beyond spec: on Anthropic, `max_tokens` is raised to
budget+2048 when thinking is on (the API requires max_tokens >
budget_tokens — the default CHAT_MAX_TOKENS=4096 would otherwise 400 at
medium/high). Escape hatch if an OpenAI-compatible endpoint rejects the
param: add "none" to reasoning.json levels (a provider 400 is surfaced as
the documented ❌ Provider error, per acceptance criteria). Tests: 5 in
`tests/test_chat_reasoning_mode.py`.
