# Issue 003.10 — Model file delivery: drop auto-attach, add `attach_file` tool + inline links (Bundle C)

- **Status**: TODO
- **PRD**: 003_PRD_deeper-assistant-integration.md (Bundle C)
- **Dependencies**: 003.08 (the download route exists)
- **Blocks**: —
- **Bundle**: C

## Goal

Remove the auto-attach-after-turn behavior (the `📁 Files from this turn:` trailing block). Add a native `attach_file_to_answer(path)` tool the model calls explicitly to deliver a file as a Chainlit download chip. Document the inline-download-link URL convention in the system prompt so the model emits relative `/api/workspace/files/workspace/<rel>` markdown links instead of hallucinating URLs.

## Why

Today every file the model creates during a turn (including scratch scripts, intermediate data, temp artifacts) is dumped to the user as a trailing attachment — noise. And because no sanctioned URL convention exists, the model hallucinated `https://workspace.simpler.com/files/…`. The model should decide which files are for the user, and have a real URL pattern for inline links.

## Scope (TDD-first)

### Failing tests first (`tests/test_chat_files_delivery.py` — new)

1. `test_attach_file_tool_spec_registered` — `files.register(toolbox)` adds a native tool `attach_file_to_answer` with the expected schema `{path: str}`.
2. `test_attach_file_tool_returns_confirmation` — calling the tool with a path that exists returns a confirmation message the model can surface; calling with a non-existent path returns a `TOOL ERROR: file not found`.
3. `test_attach_file_tool_enqueues_cl_file_element` — when executed within a Chainlit session (mocked), the tool schedules a `cl.File` element pointing at the path, to be sent at the end of the turn. (Test the scheduling mechanism, not Chainlit's send.)
4. `test_auto_attach_removed_from_on_message` — `chat/chainlit_app.py::on_message` NO LONGER calls `files.new_files_since` or sends the trailing `📁 Files from this turn:` message. (Static/inspection test: assert the symbol `new_files_since` is not called in `on_message`.)
5. `test_system_prompt_documents_inline_link_convention` — `build_system_prompt()` includes the URL convention block: teaches the model to emit relative `/api/workspace/files/workspace/<rel>` markdown links, and points to the `attach_file_to_answer` tool for rich download chips.

### Implementation

- `chat/files.py`:
  - REMOVE `snapshot_dir` + `new_files_since` usage from `chainlit_app.py::on_message` (keep the functions if other tests use them; otherwise remove dead code — decide: **remove from on_message only, keep the functions for now in case of future use**). Delete the trailing `📁 Files from this turn:` block.
  - New `register(toolbox)` (or extend existing) registering `attach_file_to_answer(path: str)`: validates the path resolves under `settings.files_dir()` (reuse the same realpath-traversal-check as the blueprint — extract to a shared helper), queues a `cl.File(name=basename, path=full_path)` to be sent at the end of the turn (collect into a session list consumed by `on_message` after `run_agent` returns), returns a model-facing confirmation.
- `chat/chainlit_app.py::on_message`:
  - Drop the `workspace_before` + `produced` dance.
  - After `run_agent` returns, flush any queued `attach_file` elements as `cl.File` chips (one message: `📁 Attached: <names>` or individual).
- System prompt update (`chat/prompts/system.md`):
  - Replace the current "any file you create… is automatically sent back" paragraph with: "To deliver a file to the user, EITHER (a) call `attach_file_to_answer(path)` for a rich download chip, OR (b) emit a relative markdown link like `[download report](/api/workspace/files/workspace/reports/report.pdf)` inline in your reply. Scratch files you create for intermediate work are NOT auto-surfaced — use one of these two methods only for files the user should receive."
- Path-check helper: extract the realpath-traversal-allowlist check from `src/routes/workspace.py` (003.08) into a shared `chat/files.py` (or `chat/workspace_security.py`) so the tool reuses it. Keep it in sync.

## Acceptance criteria

- [ ] The `attach_file_to_answer` tool produces a Chainlit download chip (rich, with name/size) when the model calls it.

## Out of scope

- File-expiry / cleanup of scratch files (the sandbox already caps resources; workspace cleanup is a future concern).
- Per-file TTL or quota.
