# Issue 003.02 — `instance/assistant/` app-data dir foundation (Bundle B/C)

- **Status**: TODO
- **PRD**: 003_PRD_deeper-assistant-integration.md (Cross-cutting)
- **Dependencies**: none
- **Blocks**: 003.03, 003.04, 003.05, 003.06, 003.07
- **Bundle**: foundation (shared by A/B/C)

## Goal

Create the `instance/assistant/` directory tree (gitignored, survives upgrades) and a single settings-access module that all other issues read from / write through.

## Why

Three workstreams (model list, editable system prompt, instance-scoped skills) all need per-instance file storage outside git. Building the dir + accessor once avoids duplication and races.

## Scope (TDD-first)

### Failing tests first (`tests/test_assistant_settings.py` — new file)

1. `test_instance_dir_created_on_access` — calling `instance_dir()` creates `instance/assistant/` (and parent) if missing; returns the path.
2. `test_system_prompt_path_returns_override_when_present` — when `instance/assistant/system.md` exists, `system_prompt_path()` returns it; when absent, returns `chat/prompts/system.md` (the shipped default).
3. `test_models_json_path` — `models_json_path()` returns `instance/assistant/models.json`; reading it when absent returns `[]` (no crash).
4. `test_skills_dir_instance_then_shipped` — `instance_skills_dir()` returns `instance/assistant/skills/`; `bundled_skills_dir()` returns `chat/skills/`; both created/exist on access.
5. `test_gitignore_entry_added` — `instance/assistant/` (and only that) is gitignored.

### Implementation

- New module `chat/assistant_settings.py` (pure path helpers + a thin JSON read/write for `models.json`):
  - `instance_dir() -> str` — `instance/assistant/`, `os.makedirs(..., exist_ok=True)`.
  - `system_prompt_path() -> str` — instance override if exists, else `chat/prompts/system.md`.
  - `models_json_path() -> str`, `read_models() -> list[str]`, `write_models(list[str])`.
  - `instance_skills_dir() -> str`, `bundled_skills_dir() -> str` (both `exist_ok=True`).
  - `reset_system_prompt()` — deletes the instance override (used by Bundle B "reset to shipped" button).
- `.gitignore`: add `instance/assistant/` (the whole subtree).
- Do NOT touch `chat/settings.py`'s existing `files_dir()` (that's the sandbox-shared volume, separate concern).

## Acceptance criteria

- [ ] All five failing tests pass.
- [ ] `instance/assistant/` exists after first access; gitignored.
- [ ] `chat/assistant_settings.py` is importable WITHOUT Chainlit's runtime (pure stdlib + `chat.settings`).
- [ ] No other module imports `assistant_settings` yet (downstream issues will).

## Out of scope

- The settings panel UI (issue 003.07).
- The model picker migration (issue 003.04).
- Writing any `models.json` content (defaults to empty list → caller falls back to env).
