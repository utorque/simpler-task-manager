Problem. The frontend/backend contract is "whatever to_dict() returns"; new fields disappear in update_task's hand-maintained if-stanza whitelist and the single-vs-array branch means the API shape literally depends on LLM output cardinality.

Solution. Introduce a thin DTO layer (Pydantic / marshmallow / hand-rolled schematic) per endpoint: one explicit shape in, one explicit shape out. to_dict becomes the DTO's serializer; update_task's whitelist becomes schema-driven validation.

  • depth: interface shrinks, implementation absorbs the wrappers
  • leverage: JS clients consume a contract, not an accident
  • Validation lives in one place — kills the 422-vs-400 uncertainty
  • Unblocks TODO.md:8 (advanced task creator) with richer payloads
Largest blast radius of the five. Conflicts with the permissive single-vs-array return shape at app.py:174-177 — sequence after candidates 1–3.
